# Cloud Deployer — документация продукта

> Точка входа для людей и ИИ-агентов. Прочитай это первым, прежде чем менять код.

Этот каталог — **источник истины** о том, что мы строим, почему, в каком оно
состоянии и куда движемся. Код меняется быстро; эти документы держим
актуальными, чтобы любой агент мог «холодно» включиться в работу.

## Что это за продукт (в одном абзаце)

Self-hosted PaaS — «мини-Kubernetes для малого бизнеса». Продукт ставится на
сервер **одной командой**, поднимает себя и инфраструктуру (reverse-proxy,
выпуск SSL), и даёт владельцу **веб-UI**, через который он заливает приложение
(ZIP/репозиторий), привязывает домен и получает работающий сервис с HTTPS — без
DevOps-инженера. Ядро держит сервисы живыми по принципу «желаемого состояния»
(как Kubernetes): сказано «нужно N реплик» — система поддерживает N.

## Бизнес-смысл

Сократить расходы малого и мало-среднего бизнеса на деплой. Вместо найма
DevOps / аренды managed-платформы — один дешёвый VPS и наш продукт.
Опора: **безопасность, надёжность, для MVP — простота**.

## Текущий статус одной строкой

**MVP в работе.** Ядро стабилизировано, стек полностью контейнеризован
(`docker compose up -d --build`), one-command install (`install.sh`/`install.ps1`)
с авто-онбордингом админа, тесты (pytest). Сетевая модель — общая `deployer-net`.
**Установка одной командой проверена на чистом Debian 11** (панель по HTTPS с
Let's Encrypt). Установка и первый деплой — [`07_DEPLOY.md`](07_DEPLOY.md).
